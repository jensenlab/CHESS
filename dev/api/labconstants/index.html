CHESSLabConstants API Reference

CHESSLabConstants is overwhelmingly data – registered reagents, organisms, location kinds, instruments, and standard stock recipes, defined via CHESSCore's registration macros (see Registering Lab Constants). The tables below are built directly from registry_summary, so they stay in sync with what's actually registered.

Hand-written functions

CHESSLabConstants.register_reagent! — Function
register_reagent!(type::DataType,lab_id::AbstractString,name::AbstractString,pubchemid::Union{Integer,Missing}=missing)

Author-time tool: look up lab_id's molecular weight/density in the local cache (chemical_cache_path); on a cache miss, fetch them from PubChem via get_mw_density (requires pubchemid) and cache the result. Returns a ready-to-paste @reagent line – it does not eval anything or write to solids.jl/liquids.jl itself. Paste the returned line into the appropriate file (alphabetized) and commit.

type must be CHESSCore.Solid or CHESSCore.Liquid. Omit pubchemid for a compound with no PubChem entry (e.g. a rich-media broth) – molecular weight/density are recorded as missing.

Example:

julia> register_reagent!(CHESSCore.Solid,"boric_acid","Boric Acid",7628)
"@reagent boric_acid \"Boric Acid\" Solid 61.84u\"g/mol\" 1.435u\"g/mL\" 7628"

CHESSLabConstants.register_organism! — Function
register_organism!(lab_id::AbstractString,genus::AbstractString,species::AbstractString,strain::AbstractString="";
                    atcc_id::Union{AbstractString,Missing}=missing,notes::AbstractString="")

Author-time tool, mirroring register_reagent!: records lab_id's genus/species/strain (plus atcc_id/notes for the record – CHESSCore.Organism itself only models genus/species/ strain, so those two are cached but not part of the returned line) in the local cache (chemical_cache_path), then returns a ready-to-paste @organism line. Paste it into organisms.jl to register it, or keep it in a private, gitignored file purely for your own records if it shouldn't be public – a private file like this is not included by the package itself, so anything kept there is never actually registered.

Example:

julia> register_organism!("SMU_UA159","Streptococcus","mutans","UA159";atcc_id="700610",notes="wild type")
"@organism SMU_UA159 \"Streptococcus\" \"mutans\" \"UA159\""
source
CHESSLabConstants.register_chemical! — Function
register_chemical!(lab_id::AbstractString,name::AbstractString,charge::Integer,pubchemid::Integer)

Author-time tool, mirroring register_reagent! for CHESSCore.Chemical identities (ions) instead of CHESSCore.Reagents: looks up lab_id's molecular weight in the local cache (chemical_cache_path); on a cache miss, fetches it from PubChem via get_mw_density (density is fetched but discarded – Chemical has no density field) and caches the result. Returns a ready-to-paste @chemical line – Chemical itself has no pubchemid field, so this is authoring convenience only, not permanent provenance. Paste the returned line into chemicals/dissociation.jl and commit.

Example:

julia> register_chemical!("Na⁺","Na+",1,923)
"@chemical Na⁺ \"Na+\" 1 22.9897693u\"g/mol\""
source
CHESSLabConstants.get_mw_density — Function
get_mw_density(cid::Integer)

Query the PubChem database for the checmical properties of the compound with PubChem ID cid,

get_mw_density returns the molecular weight (g/mol) and density (g/mL) as a Tuple.

source
CHESSLabConstants.chemical_cache_path — Constant
const chemical_cache_path

Path to the SQLite cache of PubChem-derived properties, keyed by LabID: Reagents (Name/Type/MolecularWeight/Density/CID, backing register_reagent!) and Chemicals (Name/Charge/MolecularWeight/CID, backing register_chemical!) – named to match CHESSCore.Reagent/CHESSCore.Chemical exactly, not the other way around. Used only by those two author-time tools – package load never touches this file or the network.
